# SaaS-Churn-Analysis
Customer churn analysis for a SaaS product using SQL and product insights

# Overview
This project analyzes customer churn for a fictional SaaS platform (RavenStack) to identify key drivers of churn, revenue loss, and retention opportunities.

# Objectives
- Analyze churn trends over time
- Identify top churn reasons
- Quantify revenue loss due to churn
- Detect early churn signals (downgrade behavior)
- Provide product recommendations to reduce churn

# Dataset
SaaS Subscription & Churn Analytics Dataset (Kaggle)

## 🛠 Tools Used
- SQL (BigQuery)
- Data Analysis
- Product Thinking

# INSIGHTS/KEY FINDINGS
1. Churn increased sharply over time, peaking at 96 in Dec 2024, indicating a rapidly worsening retention issue.
2. Feature gaps are the leading churn driver and contribute the highest revenue loss (~$1.9K), highlighting core product limitations.
3. Pricing concerns (“expensive”) and competitive alternatives are major factors influencing churn decisions.
4. 20.5% of users upgraded before churning, suggesting unmet expectations and poor value realization post-upgrade.
5. Low reactivation rate (~10%) indicates most churn is permanent, with limited natural recovery.

# RECOMMENDATIONS
1. Prioritize feature enhancements aligned with top churn reasons to improve product-market fit.
2. Improve post-upgrade onboarding and feature adoption to ensure users realize value after upgrading.
3. Optimize pricing strategy and value communication to address cost-related churn.
4. Strengthen competitive differentiation by addressing gaps highlighted in user feedback.
5. Launch targeted retention and win-back campaigns for high-risk and recently churned users.
